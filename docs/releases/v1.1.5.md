# v1.1.5 Release Notes

## Highlights

- Reworked the endless boss loop so the first Cluster Core appears at 10:00 and later fusion bosses appear every 3 minutes at 13:00, 16:00, 19:00, and onward.
- Added 1-minute endless boss pressure after 10:00; when the active boss cap is already full, the pending boss pressure empowers the current boss instead of stacking extra bosses on screen.
- Added boss affixes with readable interaction feedback: Devour, Drop Seal, Weapon Silence, and Control Warp.
- Added low-cost boss interaction effects, including cast rings, boss-player/item link lines, pulled-pickup markers, player debuff rings, boss healing feedback, and absorb-blocked feedback when the player intercepts pulled pickups.
- Improved 0.5x late-game enemy movement smoothness by reducing normal-enemy AI update skipping after 10:00 while the precision slow speed is active.
- Reduced mobile enemy pop-in when moving quickly toward screen edges by spawning and re-placing normal enemies farther outside the visible view, with a short fade-in for newly placed enemies.
- Improved mobile perceived FPS by refreshing dense normal-enemy mass layers more frequently at 1x and even more aggressively at 2x/3x, while keeping gameplay balance unchanged.
- Preserved active boss debuffs through crash-safe run snapshot restore, so Continue Run keeps Drop Seal, Weapon Silence, and Control Warp timers consistent.
- Added a headless browser runtime simulation covering PC/mobile 20-minute accelerated runs, 0.5x motion checks, and boss interaction smoke checks.

## Balance note

- Weapon stats, enemy stats, boss damage, XP, drop rates, and base normal enemy spawn pacing are intentionally unchanged.
- The boss schedule and boss interaction layer are gameplay additions; mobile changes target placement/readability/render cadence rather than making enemies weaker.
- If the screen already has the maximum allowed active bosses, scheduled boss pressure becomes boss empowerment to avoid mobile performance collapse from unlimited boss stacking.
