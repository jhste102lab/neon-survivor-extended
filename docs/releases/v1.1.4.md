# v1.1.4 Release Notes

## Highlights

- Added crash-safe run snapshots so an active run can be restored from the title screen with the new Continue Run button after a tab reload or browser restart.
- Hardened pause/resume restoration so recovered pause states show the correct overlay/HUD and keep audio suspended until the player resumes.
- Added a runtime-only visual performance budget that reacts to measured frame cost without changing enemy stats, spawn pacing, damage, XP, or drop balance.
- Reduced mobile late-game rendering pressure by switching dense enemy groups to the mass render layer much earlier, especially on touch/mobile viewports.
- Reduced particle and floating-text allocation churn by reusing capped visual objects and scaling non-critical burst effects under high visual pressure.
- Added adaptive particle rendering stride so overloaded frames draw fewer non-critical particles while gameplay entities remain simulated.
- Quantized glow/shape sprite cache radii to prevent many near-duplicate prerendered canvases during high-load mobile play.
- Fixed headless game-over state progression so accelerated simulations and stability tests can detect death/over transitions correctly.
- Added runtime stability regression coverage for snapshot save/restore, proof redaction, corrupt snapshot rejection, headless game-over, and visual pressure tracking.

## Balance note

- Gameplay balance is intentionally unchanged: no weapon, enemy, boss, damage, XP, drop-rate, or spawn-policy tuning was included in this release.
- The mobile changes target rendering, visual effects, cache pressure, and recovery behavior only.
